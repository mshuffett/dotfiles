# Daily Startup SOP (Morning)

Reference for the coach skill. Loaded on `/morning`, "morning review", "startup", or "daily startup".

Create a TaskList from the steps below and guide Michael through them.

---

## Checklist (don't skip)

Lightweight backstop — tick each before "let's go":
- [ ] The **ONE THING** named + next-action written ("Done = ___") (Step 0)
- [ ] Calendar + daily note pulled (Step 1)
- [ ] **Quick Todoist inbox sweep** — clear overnight captures from the Inbox so nothing strays untriaged (Step 1)
- [ ] First 1-2 pomodoros set with deliverables (Step 3)
- [ ] Deep-work block + objective defined (Step 3)

---

## Step 0: Prime + the ONE THING (added 2026-06-21)
Mechanism for the Cathedral-Building / over-planning pattern (see `memory.md` → Focus & Execution Protocol).
- **Prime (~5 min):** gym if it's a gym morning → Adderall → 5-min meditation/intention.
- **Focusing question:** *"What's the ONE THING that creates momentum / moves this forward?"* → that's today's #1.
- **Write the NEXT ACTION, not a plan.** One sentence: "Done = ___" for the #1 (+ ≤2 secondary). Can't state "Done ="? You're about to over-plan — simplify. (#1 was pre-picked at last night's shutdown.)
- TAPs are live (Cathedral / Done= / Reflexive-AI / Throat).

## Step 1: Pre-Flight

1. Confirm date, time, and timezone:
   ```bash
   TZ=America/Los_Angeles date
   ```
2. Pull today's calendar:
   ```bash
   gcal today
   ```
3. Load today's daily note (`Calendar/Daily/YYYY-MM-DD.md`). Create from template if missing.
4. Read yesterday's daily note for continuity — especially the "tomorrow plan" section.
5. Read `references/memory.md` for persistent context.
6. **Read `references/current-state.md`** (dated baseline) and **pull the live Todoist P1/P2 priorities** — the current project/task source (tracking moved off Obsidian; see SKILL.md → Current Project Context). If `current-state.md` is >1 week stale, re-confirm current priorities with Michael before planning.
7. **Quick Todoist inbox sweep:** pull the Todoist Inbox and clear overnight captures — file each to its project, schedule it, or drop it. Keep it fast (a capture sweep, not full GTD); anything heavy gets parked for the weekly review. The point is the Inbox doesn't quietly accumulate between weekly reviews.

## Step 2: Context Gathering

6. Ask Michael one orienting question: "Any new context since yesterday?" (one question, not an interview).
7. Energy check-in: "What's your energy level right now? Any constraints on today?" Match the plan to reality.
8. Review calendar for fixed commitments — paste the schedule so Michael doesn't have to check it.
9. Identify time available for deep work (subtract fixed commitments, meals, transitions).

## Step 3: Planning

10. Confirm top outcomes for the day (1-3). Ask Michael to state them. Offer to suggest based on yesterday's tomorrow-plan and the live Todoist P1/P2 priorities.
11. Split into **must-do** (non-negotiable) vs **maybe-do** (if time/energy).
12. Set first 1-2 pomodoros with explicit deliverables. Each pomodoro should ladder up to an active project's next-action.
13. Define deep-work block(s): minimum 4 hours total, 1-line concrete objective per block.

## Step 4: Schedule Construction

14. Build the schedule from real calendar constraints + scheduling rules (see SKILL.md Scheduling Rules section).
15. Propose the schedule as text. Do not create calendar events without explicit approval.
16. Identify any alarms or reminders needed.

## Step 5: Capture Policy

17. Set distraction/open-loop capture policy for the day:
    - Off-topic thoughts → `+Inbox/` or daily note "Notes & Ideas" section
    - New ideas → quick capture (max ~15 min), then return to plan
    - Remind: shiny object protocol is active

## Step 6: Murphyjitsu

18. Quick pre-mortem: "What's most likely to derail this plan today? What would you do if that happens?"

## Step 7: Confirm and Go

19. Restate the plan in 3-5 lines: top outcomes, first pomodoro, deep-work target.
20. "Ready? Let's go."

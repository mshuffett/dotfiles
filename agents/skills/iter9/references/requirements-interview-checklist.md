# Requirements Interview Checklist

Run this before creating implementation-ready tasks.

## Interview Objectives

1. Define the problem and desired user outcome.
2. Identify who the primary users are.
3. Define must-have behaviors and explicit non-goals.
4. Capture constraints (technical, compliance, timeline, dependencies).
5. Capture unknowns, assumptions, and major risks.
6. Define success criteria and acceptance expectations.

## Minimum Question Set

- What user problem are we solving?
- What must be true for this to be considered successful?
- Who are the main users and top workflows?
- What is explicitly out of scope for this slice?
- What constraints are non-negotiable?
- What decisions are already locked?
- What remains unknown and how will we de-risk it?

## Sign-Off Rule

Requirements are sign-off ready when:
- requirements list is concrete and testable
- non-goals are explicit
- unknowns are resolved or deferred with owner/date
- user confirms requirements are sufficient for the next executable slice

Record sign-off in the plan with approver + timestamp.

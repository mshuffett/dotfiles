`2-Areas/System/Active Projects.md` had a "Last updated: 2026-04-19" stamp
but by 2026-05-27 was significantly stale — it listed PE GTM / VC Strategy /
Dev Sprint / Compose AI Direction as primary, when Michael's actual frame
had pivoted to the Constitution Project, Waycraft, and network-state /
abundance / coordination / psychedelics / mind-changing AI work.

**Why:** I used the dated project list to build a fit assessment for Edge
Esmeralda and got the wrong bullseye (recommended Week 3 / AI tooling) when
the real best-fit week was Week 1 (Psychedelic Futures) + Week 4 (new-cities
network-state summit). Michael called the project list "quite out of date"
explicitly.

**How to apply:** When the project list is older than ~4 weeks, or when
recommendations are about to be made on its basis, first cross-check against:
1. Most recent daily notes (`Calendar/Daily/`)
2. Files modified in the last 2 weeks (`find ... -mtime -14`)
3. The Claude Archive Wiki Synthesis if it has a more recent date stamp
4. Recently created project folders (`1-Projects/`)
5. If still ambiguous, ASK before committing to a fit framing.

The vault is updated unevenly. CLAUDE.md flags Active Projects.md as canonical
but also instructs to ask the user to re-state active projects if the file is
stale — apply that rule by default for non-trivial fit/strategy work, even
when the file's own stamp claims freshness. [[current-thesis-may-2026]] has
the May 2026 frame.
